# Vibe Coding CodePlaybook #8

**Concept Central**
Le déploiement d'une application ou d'un site web est le processus de mise en ligne de votre codebase sur un serveur distant pour qu'il soit accessible au public. Cela implique de stocker vos fichiers, de les compiler et de les exécuter sur un serveur 24h/24.

**Points Clés**

1. **Déploiement** : Le déploiement signifie déplacer votre codebase d'un ordinateur local à un serveur distant.
2. **Hôte** : Un hôte est une plateforme qui stocke vos fichiers, compile et exécute votre code.
3. **Serveur** : Un serveur est un ordinateur distant qui exécute votre application 24h/24.
4. **Types de sites** : Il existe deux types de sites : statiques (identiques pour tous les utilisateurs) et dynamiques (générés à la volée en fonction des interactions des utilisateurs).
5. **Déploiement précoce** : Il est recommandé de déployer votre application tôt pour éviter les erreurs et les problèmes de compatibilité.

**Insights Actionnables**

1. **Choisissez un hôte adapté** : Sélectionnez un hôte qui prend en charge votre framework et vos besoins spécifiques.
2. **Configurez vos variables d'environnement** : Assurez-vous de configurer vos variables d'environnement pour éviter les erreurs de déploiement.
3. **Vérifiez vos paramètres de construction** : Vérifiez vos paramètres de construction pour vous assurer qu'ils sont corrects pour votre hôte et votre framework.
4. **Déployez tôt** : Déployez votre application tôt pour éviter les erreurs et les problèmes de compatibilité.
